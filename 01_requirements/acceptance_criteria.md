# Acceptance Criteria

## 기능 수용 기준

### SOP 실행
- Given 승인된 SOP가 등록되어 있을 때, When 작업자가 세션을 시작하면, Then 시스템은 첫 단계를 표시하고 step state를 `WAITING_FOR_EVIDENCE`로 설정한다.
- Given 단계 완료 조건이 충족되지 않았을 때, When 사용자가 다음 단계로 이동하려 하면, Then 시스템은 이동을 차단하고 누락 조건을 표시한다.
- Given 단계가 critical일 때, When AI confidence가 임계값보다 낮으면, Then 자동 통과하지 않고 수동 승인 또는 재촬영을 요구한다.

### 객체/상태 감지
- PPE 검출은 현장별 필수 PPE policy와 매칭되어야 한다.
- 라벨 검증은 OCR 텍스트, 바코드/QR, 객체 위치 중 최소 하나 이상의 증적을 저장해야 한다.
- 나사/부품 검사에서 누락 위치는 이미지 좌표 또는 작업 대상 좌표계 기준으로 표시되어야 한다.

### 관리자 대시보드
- 현장별 작업 성공률, 실패 단계 TOP 10, 오류 유형, 작업자별 재시도 횟수를 제공해야 한다.
- 리포트는 작업 세션, SOP 버전, 모델 버전, 승인자, 증적 링크를 포함해야 한다.

## 품질 기준

| 항목 | 기준 |
|---|---|
| Critical 객체 검출 recall | 고객 합의 기준 이상, 기본 0.95 목표 |
| Critical false negative | Pilot 기간 동안 0건 목표 |
| 경고 지연 | 로컬/엣지 추론 기준 2초 이내 |
| 증적 누락률 | 1% 미만 |
| 앱 크래시율 | 0.5% 미만 |
| 이벤트 동기화 성공률 | 99.5% 이상 |

## 운영 수용 기준

- 고객 테넌트별 데이터 분리 검증 완료
- SOP 버전 롤백 가능
- 모델 버전 롤백 가능
- 관리자 권한 변경 이력 보존
- 장애 시 degraded mode 문서화
