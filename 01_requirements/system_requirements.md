# System Requirements

## Functional Requirements

| ID | 요구사항 | 우선순위 |
|---|---|---|
| FR-001 | SOP를 단계, 조건, 증적, 승인 규칙으로 등록할 수 있어야 한다. | P0 |
| FR-002 | 작업자는 모바일/XR 클라이언트에서 작업 세션을 시작할 수 있어야 한다. | P0 |
| FR-003 | 시스템은 카메라 영상에서 PPE, 부품, 나사, 라벨, 위험구역을 검출해야 한다. | P0 |
| FR-004 | 시스템은 단계별 완료 조건을 자동 또는 수동 승인으로 판정해야 한다. | P0 |
| FR-005 | 순서 오류, 금지구역, PPE 미착용, 누락 상태를 실시간 경고해야 한다. | P0 |
| FR-006 | 관리자 대시보드는 작업 이력, 실패 단계, 오류 유형, 증적을 제공해야 한다. | P0 |
| FR-007 | 고위험 단계의 예외 진행은 승인자 권한과 사유 기록을 요구해야 한다. | P0 |
| FR-008 | 각 판단에는 모델 버전, 정책 버전, confidence, 입력 증적 링크가 기록되어야 한다. | P0 |
| FR-009 | API를 통해 MES/QMS/LIMS/EHS 시스템과 연동할 수 있어야 한다. | P1 |
| FR-010 | 오프라인 상태에서 작업 이벤트를 로컬 큐에 저장 후 재동기화해야 한다. | P1 |
| FR-011 | SOP 문서 업로드 시 LLM이 절차 초안을 구조화할 수 있어야 한다. | P1 |
| FR-012 | XR 글래스는 핸즈프리 음성/제스처 입력을 지원해야 한다. | P2 |

## Safety-Critical Policy

- 위험 물질, 전기, 고온, 압력, 클린룸, 고가 장비 관련 단계는 `critical=true`로 관리한다.
- Critical 단계에서 AI confidence가 임계값 이하이면 자동 통과 불가.
- Critical 단계는 현장 정책에 따라 2인 확인 또는 승인자 확인을 요구한다.
- 오류 발생 시 작업자에게 경고만 하는 것이 아니라 “정지/재확인/승인 요청” 상태로 전환한다.
