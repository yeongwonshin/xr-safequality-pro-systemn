# Support & SLA

## Support Levels

| Level | 대상 | 응답 목표 |
|---|---|---|
| L1 | 사용법, 계정, 기본 설정 | 영업일 1일 |
| L2 | 절차/대시보드/연동 문제 | 8시간 |
| L3 | 모델/엣지/성능 문제 | 4시간 |
| Sev1 | 안전 critical 장애, 감사로그 장애 | 즉시/계약별 |

## Incident Severity

| Severity | 예시 |
|---|---|
| Sev1 | critical 작업이 잘못 통과, audit write failure, 전체 서비스 중단 |
| Sev2 | 특정 현장/절차에서 작업 차단, edge gateway 장애 |
| Sev3 | 대시보드 지연, 일부 리포트 오류 |
| Sev4 | 기능 문의, 개선 요청 |

## Customer Responsibilities

- 현장 SOP 정확성 유지
- 작업자 고지와 교육
- 현장 네트워크/디바이스 관리
- 모델 개선을 위한 피드백 제공
- 법규/노사/개인정보 정책 검토

## Vendor Responsibilities

- 시스템 가용성 유지
- 보안 패치
- 모델/정책 변경 이력 관리
- 장애 대응과 root cause report
- 문서/교육 자료 제공
